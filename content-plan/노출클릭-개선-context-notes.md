# 노출·클릭 개선 — 컨텍스트 노트 (2026-07-17)

## 왜 이 작업인가
- 서치어드바이저 데이터상 브랜드 검색 노출(3,074)은 많지만 클릭(5)이 거의 없음. 검색 결과에서 스마트플레이스 카드가 상단을 차지해 홈페이지가 클릭을 못 가져가는 구조.
- 색인 문서가 홈 1개뿐이라 노출이 늘어날 재료가 없음. 반면 주차/예약/무료체험 등 목적형 검색은 CTR이 4~16배 높음 → 목적형 질문에 답하는 전용 페이지가 답.

## 핵심 결정
1. **가이드 페이지는 Tailwind를 쓰지 않고 guide/guide.css(공용 순수 CSS)로 제작.**
   이유. tailwind.css는 index/admin에서 쓰인 클래스만 빌드되어 있어, 새 페이지에서 쓰면 재빌드가 필요하고 누락 위험이 있음. 가이드 페이지는 글 중심이라 작은 전용 CSS가 더 가볍고 안전.
2. **URL은 /guide/영문-하이픈** (예: /guide/booking-price). vercel.json에 `cleanUrls: true` 추가해 .html 없는 대표 URL 사용. 네이버 가이드의 '의미 있는 경로·밑줄 금지' 준수.
3. **원고는 content-plan 기둥글 7편을 변환.** 블로그용 1인칭 표현("다녀온 후기")은 공식 사이트 톤으로 조정. 사실 정보는 단일정보원천.md 확정값 기준.
4. **네이버 블로그에 같은 글을 올릴 계획이면 그대로 복붙 금지** (유사문서 위험). 블로그용은 구조·문체를 다르게 리라이팅해서 사용 — SEO 가이드 18.5절(OSMU).
5. 홈 title에 "공식 홈페이지"를 넣은 이유. 브랜드 검색에서 플레이스·블로그와 나란히 노출될 때 공식성이 클릭 차별화 요소. description에는 플레이스 카드가 못 보여주는 것(견적서 발급·FAQ·주차 지원)을 앞세움.
6. 예약 CTA URL은 홈과 동일한 네이버 예약 링크 사용(전환 동선 유지). 페이지 목적은 "검색 유입 → 정보 → 예약".
7. FAQPage 스키마는 페이지별로 해당 페이지 FAQ만 포함(홈과 질문 중복 최소화, 완전 중복 문구 지양).

## IndexNow 자동화 (2026-07-17 추가)
- 엔드포인트 확정. `https://searchadvisor.naver.com/indexnow` (POST JSON). `api.searchadvisor.naver.com`은 접속 불가 — 블로그 글들이 잘못 안내하는 경우 있음. 유효하지 않은 키로 테스트 시 422 "Invalid key" 응답으로 동작 확인함.
- 키. `9f3011277bf86bf46669e83fb927699a` — 루트의 같은 이름 .txt 파일로 서빙. IndexNow 키는 공개 값이라 워크플로에 하드코딩해도 안전.
- 자동화. `.github/workflows/indexnow.yml` — main에 *.html 또는 sitemap.xml 변경이 push되면, 키 파일 라이브 확인(최대 5분 폴링) 후 사이트맵 전체 URL을 POST. 네이버 제출로 Bing 등에도 전파(IndexNow 규약).
- 수동 실행. GitHub Actions 탭에서 "IndexNow 색인 알림" → Run workflow (workflow_dispatch 지원).
